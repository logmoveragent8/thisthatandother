# Memory Operations Checklists

## 1) Daily Memory Intake Checklist (Capture Layer)

Use at end of day (or after major work blocks).

- [ ] Wins: What got done today? (1–5 bullets)
- [ ] Decisions: Any choices made that should be remembered later?
- [ ] Blockers: What is stuck, and why?
- [ ] Follow-ups: What needs action tomorrow/this week?
- [ ] People context: Any commitments, asks, or status updates tied to specific people?
- [ ] System learnings: Any OpenClaw/tool behavior worth noting?
- [ ] Quick tags added: Project/area labels included for searchability.
- [ ] Saved to daily note: Logged in `memory/YYYY-MM-DD.md`.

## 2) Key Preferences Update Checklist (`MEMORY.md` pass)

Use weekly (or whenever new preference signals appear).

- [ ] Communication style: Tone, brevity/detail preferences, when to ask vs decide.
- [ ] Default tools/workflows: Preferred apps, channels, model choices, command habits.
- [ ] Boundaries: External actions requiring confirmation (email, public posts, etc.).
- [ ] Timing preferences: Quiet hours, heartbeat behavior, reminder cadence.
- [ ] Decision rules: “Always/never/usually” patterns that reduce repeated questions.
- [ ] Corrected assumptions: Old preference changed? Update and date-stamp it.
- [ ] Conflict check: Remove/resolve contradictory preference notes.
- [ ] Curation pass complete: Durable preferences promoted into `MEMORY.md`.

## 3) Nightly “Promote or Drop” Checklist (Distillation Layer)

Use during nightly review to keep memory clean and useful.

- [ ] Scan today + recent notes (last 1–3 days) for durable information.
- [ ] For each item, decide: Promote / Drop / Defer.
- [ ] Promote to the right place:
  - [ ] User facts/preferences → `resources/user.md` or `MEMORY.md`
  - [ ] Behavioral patterns → `resources/tacit.md`
  - [ ] Active initiatives → `projects/`
  - [ ] Ongoing status → `areas/`
- [ ] Update index (`resources/index.md`) for newly important topics.
- [ ] Archive hygiene: Move completed projects to `archive/`.
- [ ] No duplication: Ensure promoted items aren’t copied redundantly across files.
- [ ] One-line summary: Record what was promoted and why.
